# Principes

Une contribution ne doit pas être une modification directe du dépôt principal.

Elle doit passer par une *Pull Request* GitHub.

## Pourquoi une Pull Request

Une *Pull Request* garde une trace du changement.

Elle permet à une autre personne de relire le code, les règles et les tests.

Elle aide CSSDM, CSSPI et CSSMV à comprendre ce qui change.

## Point de contrôle

Si le changement touche une règle d'affaires, la *Pull Request* doit l'expliquer.
