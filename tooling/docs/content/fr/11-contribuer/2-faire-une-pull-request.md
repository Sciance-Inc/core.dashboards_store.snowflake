# Faire une Pull Request

Une *Pull Request* propose un changement.

Elle doit être courte, claire et vérifiable.

## Étapes

1. Créer une branche avec un nom descriptif.
2. Faire une modification ciblée.
3. Lancer les validations pertinentes.
4. Ouvrir la *Pull Request* dans GitHub.
5. Décrire le contexte.
6. Décrire les tests lancés.
7. Demander une revue.

## Modèle de description

```text
Contexte :

Changement :

Validation :

Impact attendu :
```

## Point de contrôle

Une personne qui relit doit comprendre le changement sans poser la première question de base.
