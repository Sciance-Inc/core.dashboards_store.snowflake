# Cycle de changement

Un changement doit suivre un parcours clair.

## Étapes

1. Décrire la demande.
2. Identifier la règle d'affaires.
3. Modifier le code *dbt*.
4. Ajouter ou ajuster les tests.
5. Mettre la documentation à jour.
6. Ouvrir une *Pull Request*.
7. Faire relire.
8. Déployer.

## Pourquoi ce cycle existe

Il protège le rapport contre les changements non relus.

Il garde une trace des décisions.

## Point de contrôle

Une demande métier doit être reliée à une modification vérifiable.
